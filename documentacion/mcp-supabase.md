# MCP Supabase (Cursor)

Для проекта подключен MCP сервер `user-Supabase`.

## Доступные MCP tools

- `sqlToRest` - конвертирует SQL в формат запроса PostgREST.
- `postgrestRequest` - выполняет HTTP-запрос в PostgREST API.

## Как использовать

1. Держать схему таблиц актуальной в Supabase.
2. Для чтения/отладки данных использовать MCP tools из Cursor.
3. Для критичных DDL-изменений предпочитать SQL миграции в репозитории.

## Границы ответственности

- MCP: быстрые проверки/чтение/прототипирование запросов.
- Код проекта: бизнес-логика, доступы, валидация, серверные проверки.

## Безопасность

- Service role ключ только на сервере.
- Не передавать секреты в клиентский код (`NEXT_PUBLIC_*` только публичные значения).
